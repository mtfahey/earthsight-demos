=== Session: 2026-03-18 (2) ===
Summary:
- Built a script to fetch all FSC-certified certificates for Indonesia from search.fsc.org
- Reverse-engineered the FSC search API (Next.js frontend backed by ASP.NET API)
- Discovered the API uses XSRF token rotation and has a 500-result hard cap per GET query
- GET requests fail with 401 for certificate types containing slashes (FM/COC, CW/FM) due to URL encoding
- Solved both issues by using Playwright to make POST requests via the browser context, which:
  - Handles auth/XSRF automatically
  - Passes filter values (including slashes) in JSON body instead of URL params
  - Still has 500-result cap, but splitting by status + certification body gets full coverage
- Final results: 1,535/1,543 total certificates (99.5%), 259/259 FM-related (100% complete)

Key files created:
- fetch_fsc_indonesia.py — Playwright-based script to pull all FSC Indonesia data

Output files (regenerated on each run):
- fsc_indonesia_all.csv — 1,535 certificates (all types: COC, FM, FM/COC, CW/FM)
- fsc_indonesia_fm.csv — 259 FM-related certificates (91 certificate holders + 161 sites)
- fsc_indonesia_details_fm.csv — 259 FM entries with full details (area, dates, addresses)

FM data highlights:
- 192 valid, 67 terminated
- Major concessions: Perum Perhutani (1.3M ha), PT RAPP/Riaufiber (350K ha),
  PT Bina Balantak Utama (295K ha), PT Sumalindo (268K ha), PT Sarmiento (207K ha)

Technical notes:
- Requires python3.9 with playwright and chromium: pip3 install playwright && python3.9 -m playwright install chromium
- FSC search API endpoints: POST /api/v1/Certification/Search?certificateUri=false (with Content-Type: application/json-patch+json)
- Detail endpoints: GET /api/v1/Certification/Details/{id}/Header, .../Address, .../Documents
- 8 COC certificates missing due to certification bodies not listed in the filter endpoint

=== Session: 2026-03-24 (2) ===
Summary:
- Built match_rpbbi_fsc.py to cross-reference RPBBI companies/concessions against FSC certification
- Uses IDF-weighted Jaccard similarity to handle common Indonesian business words
  (JAYA, LESTARI, UTAMA, INDONESIA etc. appear in 20-70 FSC orgs each — naive matching fails)
- Two match types: PBPHH mills -> FSC COC certificates, Concessions -> FSC FM certificates
- Province cross-checking as secondary signal (boost/downgrade confidence)
- Confidence tiers: high (auto-trust), medium (quick review), low (likely false positive)
- Enriched rpbbi_extracted.csv with 8 new FSC columns (4 for mill, 4 for supplier)

Results from 10 sample PDFs:
- Mills: 7/10 FSC-certified (all high confidence, all Valid status)
- Concessions: 12/81 FSC-certified at high/medium confidence
  - Most concessions are NOT FSC certified (consistent with Earthsight findings)
- Low-confidence matches flagged separately (most are false positives from common words)

Key files:
- match_rpbbi_fsc.py — matching script (IDF-weighted fuzzy name matching)
- rpbbi_fsc_matches.csv — detailed match results with scores
- rpbbi_extracted.csv — now includes FSC columns:
  pbphh_fsc_certified, pbphh_fsc_confidence, pbphh_fsc_status, pbphh_fsc_license,
  supplier_fsc_certified, supplier_fsc_confidence, supplier_fsc_status, supplier_fsc_license

Matching approach notes:
- Normalizes names: strips PT/CV/UD, removes parenthetical notes, uppercases
- IDF weighting: rare/distinctive tokens (company-specific words) weighted higher
- Requires at least 1 distinctive token overlap for high confidence
- Province matching: boosts medium->high if provinces match, downgrades if mismatch
- Low-confidence matches shown as "No (low match)" in the CSV

Usage: python3 match_rpbbi_fsc.py rpbbi_extracted.csv fsc_indonesia_all.csv fsc_indonesia_details_fm.csv -o rpbbi_fsc_matches.csv
